## Overview

Greed provides a robust set of commands for moderating your Discord server, managing user levels, and boosting engagement. All commands use the comma (`,`) prefix by default. You must have appropriate permissions to execute them.

<Callout kind="info">
Ensure the bot has the necessary permissions like `Kick Members`, `Ban Members`, and `Manage Channels` before using moderation commands.
</Callout>

## Moderation Commands

Use these for server protection and user management.

<Tabs>
  <Tab title="Kick User" icon="user-x">

    Kick a disruptive member from the server.

    <Request tabs="cURL,Discord" show-lines="true">
```bash
curl -X POST https://discord.com/api/v10/channels/{channel}/messages \
  -H "Authorization: Bot YOUR_BOT_TOKEN" \
  -d '{"content": ",kick @username Reason for kick"}'
```

```
,kick @woeful Spamming channels
```
    </Request>

    <Response tabs="Success,Error">
```json
{
  "content": "✅ @woeful: Kicked from the server. Reason: Spamming channels",
  "embeds": [{"title": "Moderation Log"}]
}
```

```json
{
  "content": "❌ Insufficient permissions to kick @woeful",
  "embeds": []
}
```
    </Response>

  </Tab>
  <Tab title="Anti-Raid" icon="alert-triangle">

    Configure mass join protection.

    <Request show-lines="false">
```
,antiraid massjoin on --threshold 5 --do kick --lock true --punish true
```
    </Request>

    <Response>
```json
{
  "content": "✅ @woeful: Updated massjoin antiraid. Punishment set to kick, threshold set to 5, lock channels set to True, punish new members set to True"
}
```
    </Response>

  </Tab>
  <Tab title="Ban User" icon="ban">

    Permanently ban a user.

    <Request>
```
,ban @username Permanent ban for violations
```
    </Request>

  </Tab>
</Tabs>

<ParamField path="user" param-type="mention" required="true">
  The Discord user to moderate.
</ParamField>

<ParamField path="reason" param-type="string" required="false">
  Optional reason for the action.
</ParamField>

## Leveling and XP Commands

Track and reward user activity with XP systems.

<Steps>
  <Step title="Enable Leveling" icon="play">
    Turn on the XP system for your server.

````bash
,leveling enable
````

  </Step>
  <Step title="Check Leaderboard" icon="bar-chart-3">

````bash
,rank leaderboard
````

    Displays top users by XP.

  </Step>
  <Step title="Reset User XP" icon="refresh-cw">
    Reset XP for a specific user.

````bash
,leveling reset @username
````

  </Step>
</Steps>

<CodeGroup tabs="Leaderboard,XP Reset">
```bash
,rank leaderboard --top 10
```

```bash
,leveling reset @woeful --confirm
```
</CodeGroup>

## Error Handling

Greed commands include built-in error responses for common issues.

<Expandable title="Common Errors" default-open="true">

- **Missing Permissions**: Bot replies with `❌ Missing permissions for this action.`
- **Invalid Arguments**: `❌ Invalid user mention or threshold value. Use --help for syntax.`
- **Rate Limited**: `⏳ Slow down! Try again in 5 seconds.`

Always check the `--help` flag for command details:

```
,antiraid --help
```

</Expandable>

<Callout kind="tip" title="Pro Tip">
Prefix can be changed via `,prefix set !` in admin channels. Test commands in a test server first.
</Callout>
