## Prerequisites

<Callout kind="info" title="Before You Begin">
Ensure you have:
- Administrator permissions on your Discord server
- A Discord account with server ownership or high-level admin rights
- Access to the server settings for role and channel management
</Callout>

Greed requires specific permissions to manage moderation, leveling, and engagement features effectively. Review the [official invite link](https://greed.best/invite) for the full permission list before proceeding.

## Invite Greed to Your Server

Follow these steps to add Greed to your Discord server quickly.

<Steps>
  <Step title="Visit the Invite Link" icon="external-link">
    Open the [Greed invite page](https://greed.best/invite) in your browser.
  </Step>
  <Step title="Select Your Server" icon="server">
    Choose the Discord server from the dropdown where you want to add Greed.
    
    Ensure you have admin rights on the selected server.
  </Step>
  <Step title="Authorize Permissions" icon="shield">
    Review and grant the required permissions, including `Manage Channels`, `Manage Roles`, `Send Messages`, `Manage Messages`, and `Kick/Ban Members`.
    
    Click **Authorize** to complete the invitation.
  </Step>
</Steps>

## Configure Initial Settings

Once added, Greed appears in your member list. Use prefix commands (starting with `,`) for setup.

<Tabs>
  <Tab title="Anti-Raid Setup" icon="shield">
    Protect your server from raids immediately.
    
````bash
,antiraid massjoin on --threshold 5 --do kick --lock true --punish true
````

    This sets a threshold of 5 joins, kicks raiders, locks channels, and punishes new members.
  </Tab>
  <Tab title="Rules Channel" icon="book-open">
    Set up an automated rules channel.
    
````bash
,rules create #rules
````

    Greed will populate the channel with customizable server rules.
  </Tab>
  <Tab title="Leveling Enable" icon="award">
    Activate the leveling system.
    
````bash
,leveling enable
````
  </Tab>
</Tabs>

## Verify Bot Functionality

Test Greed's responsiveness with a simple command.

<CodeGroup tabs="Ping,Status">
````bash
,ping
````

Expected response: `Pong! Latency: {latency}ms`

````bash
,status
````

Expected response: Bot status and uptime information.
</CodeGroup>
